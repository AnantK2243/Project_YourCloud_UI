Node Setup Tutorial

Complete guide to setting up your storage node

1 Install the Storage Client (Requires Rust Compiler)

Linux/macOS Installation 

# Clone the repository
git clone https://github.com/AnantK2243/Project_YourCloud_Client.git --depth 1

# Navigate to the storage client directory
cd Project_YourCloud_Client

# Ensure you have Rust installed (https://www.rust-lang.org/tools/install)
# If not, install Rust using the command below:
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# Run the interactive installation script
./scripts/install.sh

# The installer will:
# 1. Build the binary from source
# 2. Install it to /usr/local/bin/yourcloud_client
# 3. Run interactive setup to configure your node
# 4. Optionally create a systemd service

What the installer will ask for:

  • Node ID: From your dashboard registration
  • Auth Token: From your dashboard registration
  • Storage Path: Where to store files (default: ~/Project_YourCloud)
  • Storage Size: Maximum storage in GB (default: 40)
  • WebSocket URL: URL for frontend - ONLY CONFIGURE IF SELFHOSTING FRONTEND (default: wss://wss.project-yourcloud.me)
  • Systemd Setup: Whether to install as a system service

2 Verify Installation

The installation script will automatically configure everything for you. If you chose systemd setup, your node will start automatically.

Manual Commands (if not using systemd) 

# Start the storage node
yourcloud_client start

# Check status
yourcloud_client status

# View help for other commands
yourcloud_client --help

Systemd Commands (if using systemd service) 

# Check service status
sudo systemctl status yourcloud-storage

# Start/stop the service
sudo systemctl start yourcloud-storage
sudo systemctl stop yourcloud-storage

3 Verify Connection

Go back to your dashboard to verify that your node appears as "online" and is ready to store files.

Troubleshooting

  • Ensure your firewall allows connections on the configured port
  • Check that the storage path exists and has proper permissions
  • Verify your Node ID and Auth Token are correctly entered
  • Check the logs for any error messages
  • If using systemd: sudo journalctl -u yourcloud-storage -f
  • If running manually: Check the terminal output for errors

Configuration File

Your configuration is automatically saved to:

~/.config/Project_YourCloud/config.toml

You can edit this file later if needed.

Systemd Service Management 

# Check service status
sudo systemctl status yourcloud-storage

# Enable/disable autostart
sudo systemctl enable yourcloud-storage
sudo systemctl disable yourcloud-storage

# Start/stop/restart service
sudo systemctl start yourcloud-storage
sudo systemctl stop yourcloud-storage
sudo systemctl restart yourcloud-storage

# View service logs
sudo journalctl -u yourcloud-storage -f
